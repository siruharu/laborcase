## 리서치 단계 추정치와의 차이

플랜 수립 시 웹 검색 기반 추정치가 여러 건 있었는데, 실제 호출로 다음 항목이 **정정**되었다.

| 항목 | 리서치 추정 | 실제 | 영향 |
|---|---|---|---|
| 근로기준법 lsId | `001769` | **`001872`** | DB 저장 전 바뀐 ID 로 수정 |
| 최저임금법 lsId | 미확인 | `000129` | 신규 확정 |
| 퇴직급여법 lsId | 미확인 | `009883` | 신규 확정 |
| 남녀고용평등법 lsId | 미확인 | `000130` | 신규 확정 |
| 파견법 MST | `206723` | `223983` | 리서치가 과거 버전을 본 것으로 보임 |

→ 분석 §P4 에서 **"영속 키는 lsId + jo + hang + ho + mok 조합"** 으로 설계한 것이 옳았다. MST 는 버전 필드로만 쓴다.

## 사용 가이드

### 본문 조회 (현행)

```bash
curl -G "https://www.law.go.kr/DRF/lawService.do" \
  --data-urlencode "OC=${OC_LAW}" \
  --data-urlencode "target=law" \
  --data-urlencode "ID=001872" \
  --data-urlencode "type=XML"
```

- `ID=lsId` 로 조회하면 **현행** 버전이 내려온다.
- `MST=...` 로 조회하면 해당 **특정 버전** 이 내려온다 (과거 스냅샷).
- `efYd=YYYYMMDD` 를 추가하면 그 시행일자 기준 버전.

### 조문 단위 조회

```bash
curl -G "https://www.law.go.kr/DRF/lawService.do" \
  --data-urlencode "OC=${OC_LAW}" \
  --data-urlencode "target=lawjosub" \
  --data-urlencode "MST=265959" \
  --data-urlencode "JO=002300" \
  --data-urlencode "type=XML"
```

- `JO` 는 조문번호를 **왼쪽 2자리 + 00 으로 붙인 뒤 6자리 zero-pad** 형식으로 보낸다.
  예: 제23조 → `23` → `2300` → `002300`
- `HANG`, `HO` 도 동일 규칙(항번호·호번호 기준).
- `MOK` 은 문자 ("가", "나", "다" ...).
